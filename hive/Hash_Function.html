Hash Function in Hive Bucketing

. Hash functions in Hive bucketing are used to evenly distribute data across a specified number of buckets.
. This helps optimize query performance and is particularly useful for tables that are frequently joined on the bucketed column.

How Hashing Works in Hive Bucketing

Let's say you have a Hive table with bucketing enabled on a column named user_id.
When a new row is inserted into this table, the hash function is applied to the user_id value.

The output of the hash function determines the bucket number for that row.
For example, if the hash value falls between 0 and (number of buckets - 1), the row goes into the corresponding bucket.

The goal is to evenly distribute data across all buckets, helping with data distribution and query performance.

Example in Hive:

To create a bucketed table in Hive, you use the CLUSTERED BY clause along with the INTO BUCKETS clause in the CREATE TABLE statement.
Here's a simple example:

    
      CREATE TABLE bucketed_table (
  column1 INT,
  column2 STRING
)
CLUSTERED BY (column1) INTO 4 BUCKETS;
In this example, data in column1 will be hashed, and rows will be distributed into 4 buckets based on the hash value of column1.
